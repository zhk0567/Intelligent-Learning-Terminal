## 1. 总体策略

- 第一优先级：补齐所有“二级页 / 子页面”内容与交互闭环，避免主页面可点但详情空白。
- 第二优先级：统一子页面视觉与组件规范（标题栏、返回、空态、加载态、错误态）。
- 第三优先级：全部子页面完成后，再做跨页面联调、埋点、性能与文案优化。

## 3. 开发优先级与节奏（以“先做完页面”为目标）

## P0（立即完成，1-2 天）

- 完成所有已配置路由对应页面的“可用版本”：
  - 页面结构完整
  - 可正常进入/返回
  - 至少包含标题、主体、基础操作控件
- 验收标准：
  - 所有入口点击后不出现“空白/崩溃/无响应”
  - 深链非法参数有可见兜底

## P1（核心内容补全，2-4 天）

- 子页面业务内容补全（非空页面）：
  - 故事详情、帖子详情、商品详情、设置子页内容模块完善
- 验收标准：
  - 每个子页面具备“加载/空态/错误”三态
  - 页面内关键按钮可点击并有反馈

## P2（统一体验，1-2 天）

- 统一子页面组件风格：
  - 顶栏样式、返回行为、间距、按钮层级、文案语气
- 验收标准：
  - 页面风格一致
  - 返回路径一致，无栈混乱

## P3（联调收尾，1-2 天）

- 全链路回归与问题清零
- 验收标准：
  - 主流程无 blocker
  - 所有子页面通过冒烟测试

## 4. 实施顺序（严格按“先完成页面”）

1. 通知页、关于页、许可页、设置子页（开发快，先全铺开）  
2. 故事详情、帖子详情（内容页核心）  
3. 商品详情及拆分页（电商链路关键）  
4. 播放器详情增强与兜底完善  
5. 音乐馆“查看更多”承接页与标签结果页  
6. 全局回归与缺陷收敛

---

## 5. 页面完成定义（DoD）

每个子页面必须满足：

- 有明确标题和返回路径
- 有加载态、空态、错误态
- 有基础交互反馈（点击、Toast/Snackbar、禁用态）
- 不依赖主页面状态即可单独进入（路由参数有效）
- 在深色模式和常见分辨率下无明显布局错乱

---

## 6. 风险与对策

- 风险：页面数量多，容易“做一半可见但不可用”
  - 对策：按 DoD 逐页验收，不通过不进入下一页
- 风险：假数据与远程数据模型不一致
  - 对策：先统一 ViewModel/UI model，再接远程
- 风险：后期统一样式返工大
  - 对策：P2 统一组件前，不新增散装样式
