# 页面截图回归基线清单

> 更新时间：2026-04-21  
> 目的：为关键页面建立统一截图基线，支持后续 UI 回归对比。

## 1. 采集范围（关键页面）

- `music_hall_home`：音乐馆首页（含搜索栏与快捷入口）
- `archive_list`：资料库列表页（默认状态）
- `archive_detail`：资料库详情页（含来源时间轴与版权标签）
- `course_list`：课程列表页（默认状态）
- `course_detail`：课程详情页（含课时/教材/师资入口）
- `mall_home_sections`：商城首页四分区
- `mall_section_list`：商城分区列表页（默认排序）
- `product_detail`：商品详情页（内容态）
- `interactive_hub`：互动主页（统计卡与最近事件）
- `interactive_compose_result`：智能编曲结果态
- `interactive_review_result`：AI 点评结果态

## 2. 采集环境基线

- 设备：Android 模拟器（推荐 Pixel 6）
- 系统：Android 14（API 34）
- 主题：深色模式关闭
- 字体缩放：100%
- 显示缩放：默认
- 网络：可联网（保证图片可加载）
- 构建：`debug` 包

## 3. 命名规范

截图统一放置于：`docs/baselines/`  
文件名格式：

`<page_key>__api34__1080x2400__v1.png`

示例：

- `music_hall_home__api34__1080x2400__v1.png`
- `interactive_review_result__api34__1080x2400__v1.png`

## 4. 采集步骤（PowerShell）

1. 构建并安装：
   - `.\gradlew :app:installDebug`
2. 逐页进入目标画面并截图（可用设备自带截图或 adb）：
   - `adb exec-out screencap -p > "docs\baselines\music_hall_home__api34__1080x2400__v1.png"`
3. 对照本清单补齐全部关键页面截图。
4. 每次 UI 重大改动后，将新截图保存为 `v2`、`v3` 并保留旧版本用于比对。

## 5. 回归判定规则

- **通过**：布局结构、间距层级、关键文案和主交互区与基线一致，仅允许小幅动态内容差异。
- **需确认**：文案更新、卡片顺序变化、图标样式变化等非功能性差异。
- **不通过**：关键按钮缺失、错位、遮挡、颜色主题明显异常、页面主区结构变化未登记。

## 6. 维护要求

- 新增关键页面时，必须同步补充本清单与对应基线图。
- 回归记录建议写入 `docs/子页面后续开发计划单.md` 的阶段回填章节。
